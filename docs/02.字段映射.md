# 02. 字段映射

PB 和 Model 之间的字段名经常不一致（如 PB 用 `Id`，Model 用 `ID`）go-pbmo 提供两种映射方式

## 方式一：struct tag（推荐）

在 Model 结构体上使用 `pbmo` tag 声明映射关系：

```go
type UserModel struct {
    ID    uint64 `pbmo:"Id"`     // Model.ID <-> PB.Id
    Name  string                // 同名字段无需映射
    Email string `pbmo:"Mail"`  // Model.Email <-> PB.Mail
    Age   int
}
```

tag 映射在首次转换时自动加载（`sync.Once` 保证线程安全且只执行一次），之后缓存

## 方式二：手动映射

通过 `WithFieldMapping` 链式调用或 `RegisterFieldMapping` 批量注册：

```go
converter := pbmo.NewBidiConverter(UserPB{}, UserModel{})

// 链式调用
converter.
    WithFieldMapping("ID", "Id").
    WithFieldMapping("Email", "Mail")

// 批量注册
converter.RegisterFieldMapping(map[string]string{
    "ID":    "Id",
    "Email": "Mail",
})
```

## 方式三：选项模式

在创建转换器时通过 `WithFieldMapping` 选项配置：

```go
converter := pbmo.NewBidiConverter(UserPB{}, UserModel{},
    pbmo.WithFieldMapping("ID", "Id"),
    pbmo.WithFieldMappings(map[string]string{
        "ID":    "Id",
        "Email": "Mail",
    }),
)
```

## 优先级

当多种方式同时使用时，优先级为：

1. **手动映射**（`WithFieldMapping` / `RegisterFieldMapping`）— 最高优先级
2. **选项映射**（`WithFieldMappings` 选项）— 在创建时加载
3. **struct tag** — 最低优先级，仅当手动映射未覆盖时生效

```go
// tag 定义 pbmo:"UserId"，但手动映射覆盖为 "ClientID"
type UserModel struct {
    UserID uint64 `pbmo:"UserId"` // 会被手动映射覆盖
}

converter := pbmo.NewBidiConverter(UserPB{}, UserModel{})
converter.WithFieldMapping("UserID", "ClientID") // 优先使用
```

## 双向映射

映射关系自动双向生效：

- `ConvertPBToModel`：使用反向映射（PB 字段名 → Model 字段名）
- `ConvertModelToPB`：使用正向映射（Model 字段名 → PB 字段名）

无需手动维护双向关系。

## 禁用 tag 映射

如果不需要 tag 自动映射，可以通过选项禁用：

```go
converter := pbmo.NewBidiConverter(UserPB{}, UserModel{},
    pbmo.WithTagMapping(false),
)
```

也可以自定义 tag 名称：

```go
converter := pbmo.NewBidiConverter(UserPB{}, UserModel{},
    pbmo.WithTagName("protobuf"), // 使用 `protobuf` tag
)
```
